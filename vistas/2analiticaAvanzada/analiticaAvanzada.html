Identificación y Comprensión de los Datos
Entrada

Base de datos en excel, archivo CSV o tabla de la base de datos con todas las variables.

Proceso

• Periodicidad de entrega de la base.
• Cantidad de registros.
• Cantidad de campañas.
• Campos que vienen en las bases de datos.
• Variables sociodemográficas y de producto.
• Evento facturable.
• Oferta comercial.
• Debilidades y fortalezas.

Salida

Dash, analíticos internos, conclusiones y retroalimentación.

Validación
Entrada

• Modelo entrenado ajustado.
• Base de datos de validación.

Proceso

• Prueba del modelo con base de datos de validación.
• Uso de Matrices de Confusión, Curva ROC.
• Evaluación de metricas: precisión, sensibilidad, especificidad, kappa, R cuadrado, p-values, etc.

Salida

Modelo entrenado definitivo para implementación.

Clasificación y Categorización
Entrada

Nueva base de datos a clasificar, predecir o pronosticar.

Proceso

• Se recibe la base de datos del cliente.
• Se depura la base de datos.
• Se aplican las reglas del modelo para categorizar, estimar o predecir la variable objetivo dependiendo el caso.
• Se entrega el modelo ya categorizado a la operación para su segmentación y marcación.

Salida

Base de datos categorizadas, predicción o pronóstisco de variable objetivo.

Implementación
Entrada

Base de datos clasificadas y/o conclusiones de modelo.

Proceso

• Se entregan las base de datos del cliente a la operación ya categorizados y/o la variable objetivo estimada con los hallazgos del modelo y sus conclusiones.
• Se establece la estrategia de segmentación y marcación junto con la operación y comand center.
• Se solcializan los resultados y hallazgos encontrados en el modelo así como su utilidad y uso.
• Se realiza una simulación financiera para determinar rentabilidad de las campañas en base a composición de registros entregados y pronósticos del modelo.

Salida

Estrategias de analítica para inicios de campaña y ejercicios de simulación financiera.

Control y Seguimiento
Entrada

Resultados de la implementación de la estrategia y gestion de la base de datos por parte de la operación.

Proceso

• Se lleva a cabo control y seguimiento de la implementación de la estrategia y gestión de las base de datos a traves de los distintos dash creados para tal fin por parte de área de reporting.
• Se indentifican posibles desviós en los resultados con respecto a lo esperado, de existir tales desviós y estos son atribuibles a la operación e implementación se llevan a cabo acciones correctivas junto con los actores implicados; operación, planificación, comand center, formación y calidad entre otros. Por otro lado, si los diesviós son atribuibles al modelo se lleva a cabo una revisón del mismo y de ser el caso se realiza un calibración del mismo.
• Se elaboran analíticos internos de rentabilidad e indicadores claves para mayor control.
• Se entregan analíticos con sus conclusiones y hallagos del control y seguimiento a la operación para su análisis y retroalimentación.

Salida

Dash, analíticos internos, conclusiones y retroalimentación.
